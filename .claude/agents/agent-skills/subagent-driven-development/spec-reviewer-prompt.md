# 规范符合性审查者提示模板

派发规范符合性审查者子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规范符合性"
  prompt: |
    你在审查实现是否匹配其规范。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建的内容

    [来自实现者报告]

    ## 关键：不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确或乐观。你必须独立验证一切。

    **不要：**
    - 相信他们实现了什么的说法
    - 信任他们关于完整性的声称
    - 接受他们对需求的解释

    **要：**
    - 阅读他们写的实际代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 他们实现了请求的所有内容吗？
    - 有他们跳过或遗漏的需求吗？
    - 他们声称某些东西有效但没有实际实现吗？

    **额外/不需要的工作：**
    - 他们构建了未请求的内容吗？
    - 他们过度工程或添加不必要功能了吗？
    - 他们添加了规范中没有的"最好有"吗？

    **误解：**
    - 他们对需求的解释与意图不同吗？
    - 他们解决了错误的问题吗？
    - 他们实现了正确功能但方式错误吗？

    **通过阅读代码验证，不是信任报告。**

    报告：
    - ✅ 规范符合（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外内容，带文件:行引用]
```
