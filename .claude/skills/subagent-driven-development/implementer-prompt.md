# 实现者子代理提示模板

派发实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N：[任务名称]"
  prompt: |
    你正在实现任务 N：[任务名称]

    ## 任务描述

    [从计划中粘贴任务的完整文本 - 不要让子代理读取文件]

    ## 背景

    [场景设定：这在哪里适配、依赖、架构背景]

    ## 开始前

    如果你有以下问题：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖或假设
    - 任务描述中不清楚的任何内容

    **现在提问。** 在开始工作前提任何担忧。

    ## 你的工作

    一旦你清楚需求：
    1. 准确实现任务指定的内容
    2. 编写测试（如果任务说遵循 TDD）
    3. 验证实现有效
    4. 提交你的工作
    5. 自审（见下文）
    6. 报告回来

    工作目录：[directory]

    **工作时：** 如果你遇到意外或不清楚的事情，**提问**。
    暂停澄清总是可以的。不要猜测或做假设。

    ## 报告前：自审

    用新眼光审查你的工作。问自己：

    **完整性：**
    - 我完全实现了规范中的所有内容吗？
    - 我遗漏了任何需求吗？
    - 有我没有处理的边缘情况吗？

    **质量：**
    - 这是我最好的工作吗？
    - 名称清晰准确（匹配事物做什么，不是它们如何工作）吗？
    - 代码干净可维护吗？

    **纪律：**
    - 我避免过度构建（YAGNI）吗？
    - 我只构建了请求的内容吗？
    - 我遵循代码库中的现有模式吗？

    **测试：**
    - 测试实际验证行为（不只是 mock 行为）吗？
    - 如果要求我遵循 TDD 了吗？
    - 测试全面吗？

    如果你在自审中发现问题，在报告前现在修复。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 你测试了什么和测试结果
    - 更改的文件
    - 自审发现（如果有）
    - 任何问题或担忧
```
